판정: **예. 위 구조는 안 맞습니다. 폐기하십시오.**
근거: Obsidian은 **Properties에 임의 필드**를 둘 수 있고, **List/Tags 타입**과 **property 검색 syntax**를 지원합니다. 또한 `tags`는 YAML 리스트와 **중첩 태그**를 지원하므로, 다주제 메모를 굳이 폴더로 쪼갤 필요가 없습니다. OpenAI MCP도 핵심은 `**search`가 관련 결과 목록을 돌려주고 `fetch`가 본문을 돌려주는 것**이지, 폴더 체계를 강제하지 않습니다. Claude MCP connector도 현재는 **tool call 중심**이므로, 분류 로직은 서버 쪽에서 통제하면 됩니다. ([Obsidian](https://obsidian.md/help/properties))
다음행동: **“폴더 분류”를 버리고, “시간축 저장 + 다중 메타데이터 분류”로 바꾸십시오.**

## 수정안

### 1) 폴더는 의미 분류를 하지 않습니다

이제 폴더는 **보관용**만 씁니다.

```text
vault/
├─ mcp_raw/
│  └─ 2026/03/
├─ memory/
│  └─ 2026/03/
└─ system/
   ├─ memory_index.sqlite3
   └─ memory_index.json
```

- `mcp_raw/` = 복붙 원문 보관
- `memory/` = 정제된 기억 저장
- `system/` = 인덱스

즉, **주제별 폴더 없음**입니다.

## 2) 분류는 frontmatter에서 다중값으로 처리

Obsidian Properties는 **임의 속성**, **List**, **Tags**를 지원하고, 검색도 property/tag 기준으로 걸 수 있습니다. 그래서 분류는 폴더가 아니라 **속성 배열**로 해야 합니다. ([Obsidian](https://obsidian.md/help/properties))

### raw note 예시

```yaml
---
note_kind: raw
id: convo-2026-03-28-001
source: chatgpt
created_at_utc: 2026-03-28T10:12:00Z
topics:
  - logistics
  - invoice
  - berth
entities:
  - DSV
  - ADNOC
projects:
  - HVDC
tags:
  - mcp/raw
  - topic/logistics
  - topic/invoice
---
```

### memory note 예시

```yaml
---
note_kind: memory
id: mem-2026-03-28-014
roles:
  - decision
  - fact
source: claude
created_at_utc: 2026-03-28T10:20:00Z
topics:
  - aggregate
  - shipment
  - planning
entities:
  - DSV
projects:
  - HVDC
confidence: 0.92
status: active
tags:
  - mcp/memory
  - role/decision
  - role/fact
  - topic/aggregate
---
```

핵심은 이겁니다.

- `memory_type: "decision"` 하나만 두지 말 것
- `roles: ["decision","fact"]`처럼 **복수 허용**
- `topics`, `entities`, `projects`도 전부 **배열**

## 3) 저장 단위도 바꿔야 합니다

당신 케이스는 한 대화 안에 주제가 여러 개 섞입니다.
그래서 저장 단위는 **대화 1개 = memory 1개**가 아니라 아래처럼 가야 합니다.

### 저장 원칙

- **원문 1개**는 `raw`로 통째 저장
- 그 안에서 꺼낸 **기억 조각**만 `memory`로 여러 개 저장
- 각 memory는 1개 폴더가 아니라 **1개 note + 다중 metadata**

예시:

- 같은 대화에서
  - 운임 결정 1개
  - DSV 담당자 사실 1개
  - 내일 확인할 todo 1개
- 이렇게 **3개 memory note**로 나눔

폴더가 아니라 **노트 단위 + 속성 단위**로 분리해야 retrieval 품질이 올라갑니다.

## 4) `save_memory` 스키마도 바꿔야 합니다

기존의 단일 `memory_type` 방식은 안 맞습니다.
아래처럼 바꾸는 게 맞습니다.

```json
{
  "title": "Voyage 71 aggregate split",
  "content": "Voyage 71은 20mm only로 유지하고 5mm는 Voyage 72로 이연.",
  "source": "chatgpt",
  "roles": ["decision", "fact"],
  "topics": ["aggregate", "shipment", "planning"],
  "entities": ["DSV"],
  "projects": ["HVDC"],
  "tags": ["role/decision", "topic/aggregate"],
  "confidence": 0.92,
  "raw_ref": "convo-2026-03-28-001"
}
```

### 필드 권장안

- `roles[]` : decision / fact / preference / todo / summary
- `topics[]` : invoice / berth / customs / OOG / storage …
- `entities[]` : DSV / ADNOC / Hitachi …
- `projects[]` : HVDC / AGI …
- `raw_ref` : 어느 원문에서 뽑았는지 연결

## 5) 검색은 폴더가 아니라 metadata + full-text

Obsidian Search는 검색어와 연산자로 좁혀 찾는 구조이고, tags와 properties도 함께 활용할 수 있습니다. 따라서 retrieval은:

- text match
- `topics[]`
- `roles[]`
- `entities[]`
- `projects[]`
- recency

이 5개를 섞어야 합니다. ([Obsidian](https://obsidian.md/help/properties))

예:

- `project:HVDC + topic:invoice + role:decision`
- `entity:DSV + role:todo`
- `topic:berth + recent`

## 6) OpenAI/Claude 쪽도 이 구조가 더 맞습니다

OpenAI MCP는 `search`가 **관련 결과 목록**, `fetch`가 **선택 항목 본문**을 주면 됩니다. 즉 내부 저장소가 폴더형일 필요가 없고, **검색 가능한 인덱스형**이면 됩니다. Claude도 MCP connector에서 tool allowlist/denylist를 걸 수 있으므로, 이 metadata 기반 인덱스를 도구로 노출하면 충분합니다. ([OpenAI 개발자](https://developers.openai.com/api/docs/mcp/))

## 최종 구조

### 채택

- 시간축 폴더
- raw / memory 분리
- frontmatter 다중값
- 검색은 index 기반

### 폐기

- `decision/`
- `fact/`
- `preference/`
- `todo/`
같은 **폴더 분류**

## 한 줄 결론

**당신 구조는 “주제별 폴더형 메모장”이 아니라, “시간축 저장 + 다대다 metadata 인덱스형 memory vault”로 가야 맞습니다.**

원하면 다음 답변에서 바로 이 수정안 기준으로
`**save_memory` JSON Schema v2**와 **검색 쿼리 규칙 v2**를 다시 짜드리겠습니다.

판정: **예. 기존 `decision / fact / preference / todo` 단일 분류 구조는 폐기해야 합니다.**
근거: Obsidian은 note 상단 **Properties/YAML**에 임의 필드를 둘 수 있고, **List**와 **Tags** 타입, 그리고 **property search syntax**를 지원합니다. 따라서 다주제 메모는 폴더가 아니라 **다중 metadata 배열**로 다루는 편이 맞습니다. OpenAI MCP도 핵심은 `search`가 **결과 목록**을, `fetch`가 **본문**을 돌려주는 구조이며, Claude MCP connector도 현재 **tool calls만 지원**하므로 분류 로직은 서버 쪽 query/parser로 처리하면 됩니다. ([Obsidian](https://obsidian.md/help/properties))
다음행동: `**save_memory`는 다중 facet 배열형 v2로 바꾸고, `search`는 단일 문자열 DSL, `fetch`는 ID 조회로 고정**하십시오.

# 왜 v2가 맞는가

Obsidian Properties는 **한 note에 여러 속성**을 둘 수 있고, 값 타입으로 **List**와 **Tags**를 지원합니다. 또 properties는 자체 **search syntax**를 지원하므로, 하나의 note를 여러 facet으로 교차 검색할 수 있습니다. 그래서 다주제 메모는 폴더 트리보다 `roles/topics/entities/projects` 같은 배열형 속성이 더 잘 맞습니다. ([Obsidian](https://obsidian.md/help/properties))

# 검색 쿼리 규칙 v2

OpenAI MCP의 `search`는 **단일 query string**을 받아 `results[]`를 반환하고, `fetch`는 **단일 id**를 받아 본문을 반환하는 구조입니다. 따라서 내부 인덱스가 다중 facet이라도, 바깥 인터페이스는 **문자열 DSL**로 맞추는 것이 좋습니다. `fetch`는 `id / title / text / url / metadata`를 돌려줘야 합니다. ([OpenAI 개발자](https://developers.openai.com/api/docs/mcp/))

## 1) search query DSL v2

형식:

```text
text:"자유어 검색" role:<값> topic:<값> entity:<값> project:<값> tag:<값> after:<YYYY-MM-DD> before:<YYYY-MM-DD> status:<값> limit:<정수>
```

## 2) 지원 필드

- `text:"..."` : 본문/제목 full-text
- `role:` : decision | fact | preference | todo | summary
- `topic:` : 다중 가능
- `entity:` : 다중 가능
- `project:` : 다중 가능
- `tag:` : 다중 가능
- `after:` / `before:` : 날짜 필터
- `status:` : active | superseded | archived
- `limit:` : 1~20

## 3) 파서 규칙

- 같은 key 반복 = **AND**
  - `topic:invoice topic:berth` = 둘 다 포함
- `text:"..."` 없으면 metadata 우선 검색
- `limit` 기본값 = `5`
- `status` 기본값 = `active`
- `after/before`는 `occurred_at` 또는 `created_at` 기준
- 없는 필드는 무시
- 잘못된 토큰은 버리지 말고 `text`로 흡수

## 4) 추천 scoring 규칙

```text
score =
  text_match * 0.45 +
  role_match * 0.10 +
  topic_match * 0.15 +
  entity_match * 0.10 +
  project_match * 0.10 +
  recency * 0.10
```

가중치는 설계안입니다.
초기에는 단순 LIKE/FTS + metadata filter로 시작하고, 나중에 embedding을 붙여도 됩니다.

# 저장 규칙 v2

## 저장 단위

- **raw 1개**
- **memory 여러 개**

즉:

- 대화 원문은 `raw`
- 그 안에서 꺼낸 결정/사실/선호/todo는 각각 `memory`
- 단, 각 memory는 `roles[]` 복수 허용

## 금지

- 폴더를 `decision/ fact/ ...`로 나누는 것
- 한 note에 unrelated topic을 과도하게 몰아넣는 것
- 원문 전체를 memory로 저장하는 것

## 권장

- 하나의 memory는 **검색 가능한 주장 1개**
- 다만 역할은 여러 개 허용
  - 예: `["decision","fact"]`
  - 예: `["todo","summary"]`

# 최종 한 줄

**v2의 핵심은 “단일 분류”가 아니라 “한 memory note에 여러 역할·주제·엔터티를 배열로 붙이는 것”입니다.**

원하면 다음 답변에서 바로 이 v2 기준으로 **FastAPI `save_memory`/`search` 파서 코드**를 내려드리겠습니다.  
  
판정: **예. 아래 코드로 바로 시작하면 됩니다.**  
근거: `save_memory`는 **Pydantic request body**로 검증하고, `search`는 **단일 query string DSL**을 파싱하는 구조가 FastAPI와 잘 맞습니다. FastAPI는 request body 검증과 Pydantic 기반 query parameter model을 공식 지원하고, OpenAI MCP는 `search`/`fetch` 결과 형식을 정해 둡니다. ([fastapi.tiangolo.com](https://fastapi.tiangolo.com/tutorial/body/?utm_source=chatgpt.com))  
다음행동: **아래 1-file 예제로 parser를 먼저 붙이고**, 저장소는 나중에 SQLite/FTS로 교체하십시오.

가정: **아래 예제는 parser 검증에 집중하기 위해 저장소를 in-memory로 구현**했습니다. SQLite/FTS는 `MemoryRepository`만 바꾸면 됩니다.

## `memory_api_v2.py`

```python
from __future__ import annotations

import re
from dataclasses import dataclass
from datetime import date, datetime, timezone
from enum import Enum
from typing import Annotated, Literal
from uuid import uuid4

from fastapi import FastAPI, HTTPException, Query
from pydantic import BaseModel, Field, field_validator, model_validator


# =========================================================
# Helpers
# =========================================================

def normalize_text(value: str) -> str:
    return re.sub(r"\s+", " ", value).strip()


def normalize_list(values: list[str] | None) -> list[str]:
    if not values:
        return []
    seen: set[str] = set()
    out: list[str] = []
    for item in values:
        v = normalize_text(str(item))
        if not v:
            continue
        key = v.casefold()
        if key in seen:
            continue
        seen.add(key)
        out.append(v)
    return out


def utcnow() -> datetime:
    return datetime.now(timezone.utc)


def new_memory_id() -> str:
    return f"mem-{utcnow().strftime('%Y%m%d-%H%M%S')}-{uuid4().hex[:6]}"


# =========================================================
# save_memory v2 schema
# =========================================================

class MemorySource(str, Enum):
    chatgpt = "chatgpt"
    claude = "claude"
    grok = "grok"
    cursor = "cursor"
    manual = "manual"


class MemoryRole(str, Enum):
    decision = "decision"
    fact = "fact"
    preference = "preference"
    todo = "todo"
    summary = "summary"


class MemoryStatus(str, Enum):
    active = "active"
    superseded = "superseded"
    archived = "archived"


class RelationType(str, Enum):
    supports = "supports"
    contradicts = "contradicts"
    follows = "follows"
    supersedes = "supersedes"
    depends_on = "depends_on"
    mentions = "mentions"


class Relation(BaseModel):
    type: RelationType
    target_id: str = Field(min_length=1, max_length=120)

    @field_validator("target_id")
    @classmethod
    def validate_target_id(cls, v: str) -> str:
        return normalize_text(v)


class SaveMemoryV2(BaseModel):
    schema_version: Literal["2.0"] = "2.0"
    title: str = Field(min_length=1, max_length=200)
    content: str = Field(min_length=1)
    source: MemorySource
    roles: list[MemoryRole] = Field(min_length=1)

    topics: list[str] = Field(default_factory=list)
    entities: list[str] = Field(default_factory=list)
    projects: list[str] = Field(default_factory=list)
    tags: list[str] = Field(default_factory=list)
    raw_refs: list[str] = Field(default_factory=list)
    relations: list[Relation] = Field(default_factory=list)

    confidence: float = Field(default=0.8, ge=0.0, le=1.0)
    status: MemoryStatus = MemoryStatus.active
    occurred_at: datetime | None = None
    due_at: datetime | None = None
    language: str = Field(default="ko", min_length=2, max_length=10)
    notes: str | None = Field(default=None, max_length=2000)

    @field_validator("title", "content", "notes", mode="before")
    @classmethod
    def normalize_text_fields(cls, v):
        if v is None:
            return v
        return normalize_text(str(v))

    @field_validator("roles", mode="before")
    @classmethod
    def normalize_roles(cls, v):
        if not isinstance(v, list):
            v = [v]
        seen: set[str] = set()
        out: list[str] = []
        for item in v:
            role = normalize_text(str(item)).lower()
            if not role:
                continue
            if role in seen:
                continue
            seen.add(role)
            out.append(role)
        return out

    @field_validator("topics", "entities", "projects", "tags", "raw_refs", mode="before")
    @classmethod
    def normalize_list_fields(cls, v):
        if v is None:
            return []
        if not isinstance(v, list):
            v = [v]
        return normalize_list([str(x) for x in v])

    @field_validator("language", mode="before")
    @classmethod
    def normalize_language(cls, v):
        return normalize_text(str(v)).lower()

    @model_validator(mode="after")
    def derive_namespaced_tags(self) -> "SaveMemoryV2":
        derived = []
        derived.extend(f"role/{r.value}" for r in self.roles)
        derived.extend(f"topic/{x}" for x in self.topics)
        derived.extend(f"entity/{x}" for x in self.entities)
        derived.extend(f"project/{x}" for x in self.projects)
        self.tags = normalize_list(self.tags + derived)
        return self


class SavedMemory(BaseModel):
    memory_id: str
    schema_version: Literal["2.0"] = "2.0"
    title: str
    content: str
    source: MemorySource
    roles: list[MemoryRole]
    topics: list[str]
    entities: list[str]
    projects: list[str]
    tags: list[str]
    raw_refs: list[str]
    relations: list[Relation]

    status: MemoryStatus
    occurred_at: datetime
    due_at: datetime | None = None
    language: str
    notes: str | None = None
    created_at: datetime

    @classmethod
    def from_payload(cls, payload: SaveMemoryV2) -> "SavedMemory":
        now = utcnow()
        return cls(
            memory_id=new_memory_id(),
            title=payload.title,
            content=payload.content,
            source=payload.source,
            roles=payload.roles,
            topics=payload.topics,
            entities=payload.entities,
            projects=payload.projects,
            tags=payload.tags,
            raw_refs=payload.raw_refs,
            relations=payload.relations,
            confidence=payload.confidence,
            status=payload.status,
            occurred_at=payload.occurred_at or now,
            due_at=payload.due_at,
            language=payload.language,
            notes=payload.notes,
            created_at=now,
            updated_at=now,
        )

TOKEN_RE = re.compile(
    r'(?P<kv>[A-Za-z_][A-Za-z0-9_-]*:"[^"]*"|[A-Za-z_][A-Za-z0-9_-]*:\S+)|(?P<quoted>"[^"]*")|(?P<bare>\S+)'
)


class SearchPlan(BaseModel):
    raw_query: str
    text_terms: list[str] = Field(default_factory=list)
    roles: list[MemoryRole] = Field(default_factory=list)
    topics: list[str] = Field(default_factory=list)
    entities: list[str] = Field(default_factory=list)
    projects: list[str] = Field(default_factory=list)
    tags: list[str] = Field(default_factory=list)
    status: MemoryStatus = MemoryStatus.active
    after: date | None = None
    before: date | None = None
    limit: int = Field(default=5, ge=1, le=20)


def parse_iso_date(value: str) -> date:
    try:
        return date.fromisoformat(value)
    except ValueError as exc:
        raise HTTPException(status_code=422, detail=f"잘못된 날짜 형식: {value}") from exc


def parse_search_query(q: str) -> SearchPlan:
    q = normalize_text(q)
    if not q:
        raise HTTPException(status_code=422, detail="빈 검색어는 허용되지 않습니다.")

    text_terms: list[str] = []
    roles: list[str] = []
    topics: list[str] = []
    entities: list[str] = []
    projects: list[str] = []
    tags: list[str] = []
    status = "active"
    after: date | None = None
    before: date | None = None
    limit = 5

    for m in TOKEN_RE.finditer(q):
        token = m.group(0)

        if m.group("kv"):
            key, raw_value = token.split(":", 1)
            key = key.strip().lower()
            value = raw_value.strip()
            if value.startswith('"') and value.endswith('"'):
                value = value[1:-1]
            value = normalize_text(value)

            try:
                if key == "text":
                    if value:
                        text_terms.append(value)
                elif key == "role":
                    roles.append(value.lower())
                elif key == "topic":
                    topics.append(value)
                elif key == "entity":
                    entities.append(value)
                elif key == "project":
                    projects.append(value)
                elif key == "tag":
                    tags.append(value)
                elif key == "status":
                    status = value.lower()
                elif key == "after":
                    after = parse_iso_date(value)
                elif key == "before":
                    before = parse_iso_date(value)
                elif key == "limit":
                    limit = max(1, min(20, int(value)))
                else:
                    # 알 수 없는 key는 text로 흡수
                    text_terms.append(f"{key}:{value}")
            except ValueError as exc:
                raise HTTPException(status_code=422, detail=f"잘못된 토큰 값: {token}") from exc

        elif m.group("quoted"):
            value = token[1:-1]
            value = normalize_text(value)
            if value:
                text_terms.append(value)

        else:
            value = normalize_text(token)
            if value:
                text_terms.append(value)

    plan = SearchPlan(
        raw_query=q,
        text_terms=normalize_list(text_terms),
        roles=roles,     # Pydantic이 Enum 변환
        topics=topics,
        entities=entities,
        projects=projects,
        tags=tags,
        status=status,   # Pydantic이 Enum 변환
        after=after,
        before=before,
        limit=limit,
    )

    plan.topics = normalize_list(plan.topics)
    plan.entities = normalize_list(plan.entities)
    plan.projects = normalize_list(plan.projects)
    plan.tags = normalize_list(plan.tags)
    return plan

class SearchHit(BaseModel):
    memory_id: str
    title: str
    content_preview: str
    score: float
    metadata: dict


class SearchResponse(BaseModel):
    parsed_query: SearchPlan
    count: int
    results: list[SearchHit]


class FetchResponse(BaseModel):
    id: str
    title: str
    text: str
    url: str
    metadata: dict


@dataclass
class MemoryRepository:
    items: list[SavedMemory]

    def save(self, payload: SaveMemoryV2) -> SavedMemory:
        item = SavedMemory.from_payload(payload)
        self.items.append(item)
        return item

    def get(self, memory_id: str) -> SavedMemory | None:
        for item in self.items:
            if item.memory_id == memory_id:
                return item
        return None

    def search(self, plan: SearchPlan) -> list[SearchHit]:
        hits: list[SearchHit] = []

        for item in self.items:
            if item.status != plan.status:
                continue

            item_day = item.occurred_at.date()
            if plan.after and item_day < plan.after:
                continue
            if plan.before and item_day > plan.before:
                continue

            role_set = {r.value for r in item.roles}
            topic_set = {x.casefold() for x in item.topics}
            entity_set = {x.casefold() for x in item.entities}
            project_set = {x.casefold() for x in item.projects}
            tag_set = {x.casefold() for x in item.tags}

            if plan.roles and not all(r.value in role_set for r in plan.roles):
                continue
            if plan.topics and not all(x.casefold() in topic_set for x in plan.topics):
                continue
            if plan.entities and not all(x.casefold() in entity_set for x in plan.entities):
                continue
            if plan.projects and not all(x.casefold() in project_set for x in plan.projects):
                continue
            if plan.tags and not all(x.casefold() in tag_set for x in plan.tags):
                continue

            searchable = " ".join(
                [
                    item.title,
                    item.content,
                    " ".join(item.topics),
                    " ".join(item.entities),
                    " ".join(item.projects),
                    " ".join(item.tags),
                ]
            ).casefold()

            text_score = 0.0
            if plan.text_terms:
                matched_all = True
                for term in plan.text_terms:
                    t = term.casefold()
                    if t in searchable:
                        text_score += 3.0
                    else:
                        matched_all = False
                        break
                if not matched_all:
                    continue

            facet_score = 0.0
            facet_score += len(plan.roles) * 1.0
            facet_score += len(plan.topics) * 0.8
            facet_score += len(plan.entities) * 0.7
            facet_score += len(plan.projects) * 0.7
            facet_score += len(plan.tags) * 0.5

            age_days = max((utcnow().date() - item.occurred_at.date()).days, 0)
            recency_score = max(0.0, 2.0 - min(age_days / 30.0, 2.0))

            score = text_score + facet_score + recency_score + item.confidence

            hits.append(
                SearchHit(
                    memory_id=item.memory_id,
                    title=item.title,
                    content_preview=item.content[:240],
                    score=round(score, 4),
                    metadata={
                        "roles": [r.value for r in item.roles],
                        "topics": item.topics,
                        "entities": item.entities,
                        "projects": item.projects,
                        "tags": item.tags,
                        "confidence": item.confidence,
                        "status": item.status.value,
                        "occurred_at": item.occurred_at.isoformat(),
                    },
                )
            )

        hits.sort(key=lambda x: x.score, reverse=True)
        return hits[: plan.limit]

app = FastAPI(title="memory-api-v2")


@app.post("/save_memory", response_model=SaveMemoryResponse)
async def save_memory(payload: SaveMemoryV2):
    saved = repo.save(payload)
    return SaveMemoryResponse(
        status="saved",
        memory_id=saved.memory_id,
        metadata={
            "roles": [r.value for r in saved.roles],
            "topics": saved.topics,
            "entities": saved.entities,
            "projects": saved.projects,
            "tags": saved.tags,
            "status": saved.status.value,
            "occurred_at": saved.occurred_at.isoformat(),
        },
    )


@app.get("/search", response_model=SearchResponse)
async def search_memories(
    q: Annotated[
        str,
        Query(
            min_length=1,
            description='예: text:"aggregate split" role:decision project:HVDC entity:DSV limit:5'
        ),
    ]
):
    plan = parse_search_query(q)
    results = repo.search(plan)
    return SearchResponse(parsed_query=plan, count=len(results), results=results)


@app.get("/fetch/{memory_id}", response_model=FetchResponse)
async def fetch_memory(memory_id: str):
    item = repo.get(memory_id)
    if not item:
        raise HTTPException(status_code=404, detail="memory not found")

    return FetchResponse(
        id=item.memory_id,
        title=item.title,
        text=item.content,
        url=f"obsidian://open?vault=MemoryVault&file=memory/{item.occurred_at.strftime('%Y/%m')}/{item.memory_id}.md",
        metadata={
            "roles": [r.value for r in item.roles],
            "topics": item.topics,
            "entities": item.entities,
            "projects": item.projects,
            "tags": item.tags,
            "confidence": item.confidence,
            "status": item.status.value,
            "raw_refs": item.raw_refs,
        },
    )
```

## 핵심만 정리

- `save_memory`는 **다중 facet 저장**
- `search`는 **단일 DSL 문자열 파싱**
- `fetch`는 **ID 기반 본문 반환**
- 나중에 SQLite/FTS로 바꿀 때도 **parser와 API 계약은 그대로 유지**
