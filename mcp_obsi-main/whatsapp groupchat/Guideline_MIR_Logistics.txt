# Guideline_MIR_Logistics

## 1) Executive Summary

* `MIR..Logistics`는 Mirfa site 기준의 **delivery execution / unloading control / site receiving readiness / backload coordination** 채널입니다.
* 초기에는 **HE box / Layer / Serpentine** 중심의 전용 delivery room이었으나, 2025-08-19 이후 **Mirfa 현장 전체 logistics control room**으로 확장되었습니다.
* 반복 업무는 **daily delivery schedule, trailer arrival, unloading ongoing/completed, case no./pkg control, CICPA gate entry, DN/PL follow-up, transformer oil + empty tank return, indoor material handling, backload collection**입니다.
* 반복 리스크는 **sequence mismatch, site unloading capacity 부족, forklift/crane 공유, CICPA gate hold, fog/rain delay, indoor warehouse full, customs serial number visibility 이슈**입니다.

### 3.3 반복 Pain Point

1. **DSV 로딩계획과 현장 unloading 현실 간 불일치**
2. **트레일러 동시 도착 vs site one-by-one unloading 한계**
3. **forklift/crane이 electrical / mechanical / architecture team과 공유되어 지연 발생**
4. **CICPA gate entry hold 및 driver phone off**
5. **wrong loading / mixed cargo / MIR-SHU segregation 오류**
6. **transformer oil는 반입되지만 empty tank backload timing이 항상 맞지 않음**
7. **indoor material 증가로 site warehouse full**
8. **fog / rain / road restriction으로 ETA 흔들림**
9. **Saudi temporary export용 serial number visibility 미확보**
10. **DN / PL / driver details / pass 미리 공유되지 않아 현장 준비 차질**

## 5) Tailored Rules

### 5.1 채널 목적

이 그룹은 Mirfa site 기준 아래 5개만 다룹니다.

1. Delivery plan
2. Arrival / unloading status
3. Gate / pass / driver / document coordination
4. Storage / backload / indoor hold
5. OOG / special cargo / customs-related exception

### 5.2 태그 규칙

첫 줄 태그 고정:
`[URGENT][ACTION][FYI][ETA][COST][GATE][CRANE][MANIFEST][RISK]`

MIR 전용 권장 조합:

* trailer plan: `[ETA][ACTION]`
* gate pass / CICPA: `[GATE][ETA]`
* crane/forklift shortage: `[CRANE][RISK]`
* indoor full / hold: `[RISK][ACTION]`
* DN/PL/driver details: `[MANIFEST][ACTION]`
* wrong loading / sequence error: `[URGENT][RISK]`

### 5.3 첫줄 포맷

반드시 아래 중 하나 사용:

* `[MIR]/[ITEM]/[ETA]/[ACTION]`
* `[MIR]/[TRAILER QTY]/[UNLOADING]/[STATUS]`
* `[MIR]/[DOC]/[REF]/[REQUEST]`
* `[MIR]/[WAREHOUSE]/[SPACE]/[DECISION]`

예시:

* `[MIR]/[HE-0512]/[ETA 08:30 GST]/[UNLOADING PLAN]`
* `[MIR]/[5 TRAILERS]/[UNLOADING]/[ONGOING]`
* `[MIR]/[CICPA PASS]/[DRIVER 84326]/[REQUEST]`
* `[MIR]/[INDOOR MATERIAL]/[FULL]/[HOLD AT DSV]`

### 5.4 Delivery Plan 규칙

* **D-1 16:00**까지 다음날 delivery plan 공유.
* plan에는 반드시 아래 포함:

  * item / ref no. / case no.
  * trailer qty
  * trailer type(FB/LB/1t/3t)
  * loading date/time
  * delivery date/time
  * driver details
  * CICPA pass status
  * unloading equipment readiness
  * backload 여부
* `loading today / delivery same day`인지 `PM loading / next AM delivery`인지 반드시 구분.
* WhatsApp만으로 애매하면 반드시 **email + excel receiving status** 병행.

### 5.5 Arrival / Unloading 보고 규칙

도착 후 아래 4단계로 보고:

1. `Arrived at gate / site`
2. `Unloading ongoing`
3. `Completed`
4. `Trailer release / backload status`

필수 항목:

* time
* trailer qty
* item / pkg
* pending balance
* issue
* next action

예시:
`Arrived 08:30 / 5 trailers / unloading ongoing / forklift with electrical team / resume 10:00`

### 5.6 현실 용량 기준

* 현장은 **one-by-one unloading**이 기본.
* 동시 다발 트레일러 투입 시에도 `all release before lunch`를 기본 가정으로 잡지 않는다.
* site는 unloading 현실을 우선하며, DSV는 trailer rotation risk를 사전 공유한다.
* 불필요한 대기비용 방지를 위해, 현장 capability를 초과하는 배차는 금지.

### 5.7 Gate / CICPA 규칙

* CICPA gate 진입건은 전일 또는 당일 조기 시간에 아래 선공유:

  * driver name
  * vehicle no.
  * valid CICPA pass
  * cargo item
  * ETA to gate
  * unloading support readiness
* gate outside hold 발생 시 즉시 `[GATE][RISK]`.
* driver 연락두절 / switch off 발생 시 10분 내 DSV focal escalation.

### 5.8 장비 규칙

* crane / forklift / rigger / flagman 없으면 delivery proceed 금지.
* electrical/mechanical/architecture team과 공유 중일 경우:

  * 현재 사용팀
  * 예상 반환시간
  * unloading 재개시간
    을 함께 게시.
* breakdown / no operator / sick leave 발생 시 즉시 대체계획 공지.

### 5.9 Transformer Oil / Empty Tank

* transformer oil 반입 시 반드시:

  * qty
  * unloading status
  * same trailer backload 여부
  * empty tank balance
    를 함께 공유.
* same trailer backload가 안 되면 `empty tank backlog`로 별도 관리.
* port standby / storage cost 이슈가 있으면 `[COST][RISK]`.

### 5.10 Indoor / Warehouse 규칙

* site indoor full 시 **Mirfa indoor materials는 DSV indoor warehouse hold**를 기본 옵션으로 둔다.
* 예외:

  * panel direct to site
  * urgent installation item
* outdoor 보관 시 필수:

  * tarpaulin cover
  * item suitability 확인
  * rain/leak 사진
* warehouse capacity alert는 아래 형식:

  * indoor with A/C %
  * indoor without A/C %
  * access blockage risk
  * accept / partial / hold 판단

### 5.11 OOG / Heavy Cargo / Special Operation

적용 대상:

* cable drum
* subsea cable
* EDG cargo
* fuel tank
* aux. transformer
* Wilo pump
* pump unit / treatment unit
* scissor lift
* spare drums

필수 사전공지:

* crane / SPMT / riggers / escort
* foundation / unloading area readiness
* police escort / road permit / CICPA captain coordination
* weather check
* photo record plan

### 5.12 Mixed Cargo / MIR-SHU 분리

* MIR / SHU mixed load 금지 원칙.
* unavoidable 시:

  * pallet color code
  * strap color
  * quantity split
  * first drop / second drop
    를 명확히 표기.
* `blue straps=MIR`, `yellow straps=SHU`와 같은 구분은 사진과 함께 공지.

### 5.13 Customs / Serial Number / Export

* Saudi temporary export 또는 border customs 대상 화물은 선적 전 확인:

  * serial number visible 여부
  * temporary window / engraving / taped tag / alternate marking
  * customs acceptance risk
* packing modification은 vendor / DSV / site가 합의한 후 시행.
* `no guarantee temporary export` 조건이면 `[RISK][MANIFEST]`로 별도 공지.

### 5.14 야간/악천후 규칙

* fog / rain / flooding 발생 시 즉시 `[RISK][ETA]`.
* Mirfa 현장 heavy rain 시:

  * warehouse leak
  * laydown flooding
  * material wet risk
  * covering status
    를 1시간 내 공유.
* off-day / limited subcontractor hours와 충돌 시 same-day delivery 강행 금지.
