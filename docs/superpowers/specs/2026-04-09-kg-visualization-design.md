# 지식 그래프 시각화 대시보드 설계 명세서 (Knowledge Graph Visualization Dashboard Design)

## 1. 목적 (Objective)
본 문서는 물류 및 HVDC 프로젝트에서 추출된 지식 그래프(Knowledge Graph) 데이터를 직관적으로 탐색하고 분석하기 위한 시각화 대시보드의 아키텍처 및 UI/UX 설계 방향을 정의합니다. 복잡한 물류 이슈, 화물 추적, 창고 현황 등의 관계성을 파악하여 신속한 의사결정을 지원하는 것을 목표로 합니다.

## 2. 아키텍처 설계 (Architecture)

### 2.1 Phase 1: 인메모리 브라우저 기반 고속 시각화 (1안)
초기 단계에서는 서버 유지 비용을 최소화하고 렌더링 속도를 극대화하기 위해 클라이언트 사이드(In-memory) 렌더링 방식을 채택합니다.
- **데이터 변환**: Python 스크립트를 활용하여 백엔드에서 TTL(Turtle) 형식의 그래프 데이터를 브라우저가 소비하기 쉬운 JSON 형식으로 사전 변환합니다.
- **시각화 엔진**: 대규모 노드와 엣지를 지연 없이 처리할 수 있는 고성능 WebGL/Canvas 기반 라이브러리(예: Cosmograph 또는 Cytoscape.js)를 사용하여 React 기반의 웹 애플리케이션으로 구현합니다.
- **특징**: 별도의 무거운 그래프 데이터베이스(Graph DB)나 백엔드 서버 쿼리 없이, 정적 파일(Static JSON)을 브라우저에 로드하여 클라이언트 자원만으로 고속 필터링 및 렌더링을 수행합니다.

### 2.2 Phase 2: 에이전트 기반 대화형 그래프 인터페이스 (2안)
향후 고도화 단계에서는 사용자의 자연어 질의를 기반으로 동적인 그래프 탐색이 가능한 Agentic Chat-with-Graph 인터페이스로 진화합니다.
- **자연어-SPARQL 변환**: LLM(대형 언어 모델) 에이전트가 사용자의 자연어 질문("최근 지연된 선박과 관련된 화물 이슈를 보여줘")을 이해하고, 이를 SPARQL 쿼리로 자동 변환합니다.
- **미니 그래프(Mini-graph) 렌더링**: 전체 그래프가 아닌 질의에 해당하는 핵심 서브그래프(Subgraph) 구조만을 추출하여 채팅창 또는 동적 패널에 렌더링합니다.
- **상호작용**: 대화 맥락을 유지하면서 노드를 확장(Expand)하거나 추가적인 질문을 통해 그래프를 심층 탐색할 수 있습니다.

## 3. 데이터 흐름 (Data Flow)

Phase 1 시스템의 데이터 처리 파이프라인은 다음과 같습니다.

1. **지식 추출 및 저장**: 텍스트, WhatsApp 로그, 엑셀 문서 등에서 추출된 지식 그래프가 `.ttl` 파일 형태로 저장됩니다.
2. **데이터 파싱 및 정제**: Python 기반의 배치(Batch) 스크립트가 `.ttl` 파일을 읽어들여 시각화에 필요한 메타데이터(노드 타입, 속성, 엣지 관계)를 추출합니다.
3. **JSON 에셋 생성**: 파싱된 데이터는 클라이언트 앱이 사용할 수 있도록 두 개의 정적 에셋 파일로 분리되어 생성됩니다.
   - `nodes.json`: 노드 ID, 레이블, 타입(Type), 추가 속성(Property) 정보 포함
   - `edges.json`: Source 노드 ID, Target 노드 ID, 관계 타입(Relation) 정보 포함
4. **브라우저 로드 및 렌더링**: React 기반 대시보드 접근 시 `nodes.json`과 `edges.json`을 비동기 호출(Fetch)로 메모리에 로드한 후, 시각화 라이브러리를 통해 화면에 렌더링합니다.

## 4. UI/UX 디자인 (UI/UX Design)

### 4.1 컬러 코딩 (Color Coding)
직관적인 노드 인식을 위해 엔티티 타입별로 명확한 컬러 코딩을 적용합니다.
- 🔴 **LogisticsIssue (물류 이슈)**: Red (가장 높은 주의를 요하는 이슈 발생 노드)
- 🔵 **Shipment (화물/선적)**: Blue
- 🟠 **Vessel (선박/운송수단)**: Orange
- 🟢 **Site/Warehouse (현장 및 창고)**: Green
- ⚪ **기타 (Vendor, Person 등)**: Gray 계열 및 기타 보조 색상

### 4.2 레이아웃 및 필터링 패널 (Left Panel)
화면 좌측에는 복잡한 그래프를 제어하기 위한 필터링 패널을 배치합니다.
- **이슈 중심 필터 (Focus Mode)**: "이슈(LogisticsIssue)가 연결된 노드만 보기" 토글을 제공하여 문제 해결에 집중할 수 있도록 지원합니다.
- **엔티티 필터**: Vendor(업체), Vessel(선박), Date(날짜) 단위로 노드를 필터링하여 불필요한 데이터를 화면에서 숨깁니다.
- **검색 바**: 노드의 이름이나 ID로 특정 엔티티를 빠르게 검색하고 하이라이트 합니다.

### 4.3 인터랙션 (Interaction)
- **노드 클릭 이벤트**: 특정 노드를 클릭하면 우측 팝업이나 오버레이 패널을 통해 노드의 상세 정보가 표시됩니다.
- **Markdown 연동**: 특히 빨간색 이슈(LogisticsIssue) 노드 클릭 시, 해당 이슈의 근거가 되는 원본 Markdown 파일(예: WhatsApp 로그 또는 회의록)로 바로 이동할 수 있는 링크를 제공하여 심층 확인을 돕습니다.
- **줌 및 팬 (Zoom & Pan)**: 캔버스 내 마우스 휠 및 드래그를 통해 자유로운 화면 이동과 확대/축소를 지원합니다.

## 5. 향후 로드맵 (Future Roadmap)
- **단기 (Short-term)**: Phase 1 아키텍처를 기반으로 Python 파서와 클라이언트 시각화 뷰어 구현 및 배포.
- **중기 (Mid-term)**: 시각화 대시보드 내에 검색 및 필터링 조건 저장 기능 추가, 주기적인 TTL 자동 동기화 파이프라인 구축.
- **장기 (Long-term)**: Phase 2 아키텍처 도입. LLM(Gemma 4 등)과 통합된 채팅창 UI를 추가하여 대화형 질의를 통한 서브그래프 시각화 및 인사이트 도출 기능 완성.
