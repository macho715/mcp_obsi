# Guideline_Abu_Dhabi_Logistics

## 1) Executive Summary

* `Abu Dhabi Logistics`는 UAE HVDC 현장의 **중앙 물류 통제방**입니다. 단일 site 운영방이 아니라 **DAS / AGI / MOSB / MW4 / Port / Vessel / Yard / Gate pass / Crane / FLIFT / Inspection**을 한 채널에서 통합 조정합니다. 
* 실제 운용은 `delivery plan + vessel movement + yard control + gate pass + equipment coordination + document/inspection follow-up`의 6축으로 움직입니다. 
* 반복 리스크는 **FLIFT 지연, crane availability mismatch, high tide / harbor restriction, TPI/TUV/inspection hold, HCS damage, yard congestion**입니다. 
* 따라서 이 그룹은 단순 보고방이 아니라 **project-wide dispatch control room**으로 정의하는 것이 맞습니다.

## 5) Tailored Rules

### 5.1 채널 목적

이 그룹은 아래 6개만 다룹니다.

1. Daily delivery plan
2. Vessel movement SITREP
3. Crane / FLIFT / lifting team coordination
4. Gate pass / exit pass / entry control
5. Inspection / certificate / document follow-up
6. Cross-site priority escalation

### 5.2 태그 규칙

첫 줄 태그 고정:
`[URGENT][ACTION][FYI][ETA][COST][GATE][CRANE][MANIFEST][RISK]`

Abu Dhabi Logistics 권장 조합:

* 일일 delivery list: `[ETA][ACTION]`
* 선박 현황: `[FYI][ETA]`
* 장비 부족: `[CRANE][RISK]`
* gate issue: `[GATE][URGENT]`
* HCS crack / unsafe: `[RISK][URGENT]`
* inspection / TPI: `[MANIFEST][ACTION]`

### 5.3 첫줄 포맷

* `[ABU]/[SITE]/[ITEM]/[ETA]/[ACTION]`
* `[ABU]/[VESSEL]/[ETA-ETD]/[STATUS]`
* `[ABU]/[EQUIPMENT]/[SITE]/[REQUEST]`
* `[ABU]/[DOC]/[REF]/[ACTION]`

예시:

* `[ABU]/[DAS]/[UPC HCS]/[2025-08-29 10:30 GST]/[OFFLOADING PLAN]`
* `[ABU]/[THURAYA]/[ETA 19:00 GST]/[LOADING STATUS]`
* `[ABU]/[FLIFT 16T]/[MOSB LD]/[URGENT REQUEST]`

### 5.4 Daily delivery plan 규칙

* **D-1 16:00**까지 다음날 delivery plan 공유.
* 반드시 포함:

  * site
  * supplier
  * qty / trailer qty
  * cargo type
  * ETA
  * required crane / forklift / manpower
  * inspection / TPI / gate pass status
* 오전 plan 변경 시 `R1, R2` revision 표기.
* `today / tomorrow / afternoon` 같은 표현만 쓰지 말고 **절대시간** 병기.

### 5.5 Vessel SITREP 규칙

* 최소 **07:30 / 16:00** 2회 공유.
* 포맷:

  * Vessel
  * current location
  * ETA / ETD
  * cargo
  * constraint(tide / berth / weather / bunkering / permission)
  * next action
* 대상 선박:

  * JPT62
  * JPT71
  * Bushra
  * Thuraya
  * Taibah
  * Razan
  * 기타 대체 LCT

### 5.6 FLIFT / Crane 에스컬레이션 규칙

* 10분 무응답: `[RISK]`
* 20분 경과: 1차 escalation
* 30분 경과: 2차 escalation + 작업순서 재배치
* 반복 지연 3회 이상: vendor complaint email same day 발송

근거: 기존 guideline draft에서도 10·20·30 Rule이 이미 정의되어 있고, 실제 채팅에서도 forklift 지연이 반복됩니다.  

### 5.7 HCS / Cladding / Heavy cargo 규칙

* crack, side damage, unsafe lifting 발견 시 즉시 stop.
* 사진 다각도 + DN / bundle / trailer no. 확보.
* lifting team reject 시 현장 판단으로 강행 금지.
* cladding / HCS / bottom shell plate는 **사전 lifting method 합의** 후 진행.
* 필요 시 load test video 기록.

### 5.8 Port / Al Jaber 정책

* Al Jaber는 **A-Frame 우선** 원칙.
* 일반 RORO/FB 필요 시 dispensation 선확보.
* tide / harbor / berth 제약은 plan 확정 전에 반영.
* vessel / port limitation은 WhatsApp뿐 아니라 email 근거도 남김.

### 5.9 Gate pass / Exit pass 규칙

* gate pass는 **차량 도착 전** 완료 원칙.
* exit pass 메시지 필수 항목:

  * trailer no.
  * cntr / basket no.
  * cargo description
  * full / empty
  * destination
* mismatch 발견 시 현장 출차 중지 후 즉시 수정.

### 5.10 Yard capacity 규칙

* `NO BL SHIFTED TO LD` 또는 LD full 상태면 신규 delivery 승인 금지.
* DAS / AGI / MIR / SHU 반입 전, Mr Jeong 또는 Roy 확인 후 진행.
* 공간 부족 시:

  * hold at supplier
  * partial delivery
  * next shift / next day reschedule
    중 하나를 즉시 결정.

### 5.11 Certificate / Inspection 규칙

* TPI / TUV / WELLS / sling color code / Mulkiya 만료 전 사전확인.
* 증빙 미수령 상태에서는:

  * inspection date
  * blocking reason
  * owner
  * expected clearance time
    를 함께 게시.
* OTP, 카드, 민감정보는 그룹 직접 공유 금지.

### 5.12 Cross-site 조정 규칙

* Abu Dhabi Logistics는 실행방이 아니라 **우선순위·자원배분 방** 성격도 강함.
* SHU / MIR / DAS / AGI 관련 지시는 이 그룹에서 정하고, site execution은 각 site room으로 넘긴다.
* 따라서 이 방에는 반드시:

  * owner
  * site
  * deadline
  * next action
    을 함께 남긴다.
