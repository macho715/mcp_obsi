## 3) Observed Patterns

### 3.1 핵심 운영 패턴

| No | 패턴                                 | 내용                                                             |
| -- | ---------------------------------- | -------------------------------------------------------------- |
| 1  | MW4 loading control                | truck arrival, loading start/finish, shovel/wheel loader 상태 공유 |
| 2  | AGI West Harbor Jetty#3 offloading | berth 확보 후 offloading, high tide 시 중단·익일 재개                    |
| 3  | Tide-window execution              | morning window 중심, tide rise 후 suspension 빈번                   |
| 4  | Captain direct coordination        | ETA/ETD, draft, FW, bunkering, manifest, berth 직접 질의응답         |
| 5  | Special voyage handling            | NFP(HCS), DAS temporary jetty, ETI soil shifting 등 별도 관리       |
| 6  | Cargo-type switching               | aggregate ↔ jumbo bag ↔ HCS ↔ A-frame 간 항차 전환 빈번               |
| 7  | Harbor/anchorage dependency        | berth congestion, acid stimulation, harbor closure 영향 큼        |



### 3.2 반복 핵심 토픽

프로파일 기준 상위 토픽은 `Loading/Offloading progress`, `Delivery plan / ETA/ETD`, `AGI Harbor ops`, `Weather / Tide`, `Aggregate / Sand`, `Jumbo bags / Cement`입니다. 

### 3.3 Raw chat에서 직접 확인되는 운영 특징

* AGI offloading은 **West Harbor Jetty#3**가 반복 중심입니다. 
* `high tide`, `low tide`, `berth congestion`, `acid stimulation`, `harbor closure` 때문에 **1일 내 완료 실패 → 다음날 morning resume**가 매우 자주 발생합니다. 
* Jumbo bag 항차에서는 **ALS 145T crane**, **MW4 10T forklift**, **tarpaulin cover**, **telehandler/Forklift 논의**가 반복됩니다. 
* HCS/Wall Panel/STSR 항차는 **NFP berth, pilot, gate pass, crane availability, pre-sling, height/draft** 이슈가 중요합니다. 
* 선박 측 요청은 주로 **fresh water, diesel oil, food provision, final manifest, pre-arrival notice, CEP/vetting**입니다.

## 5) Tailored Rules

아래 규칙은 **관측 패턴을 기준으로 설계한 운영 rule**입니다. 사실 서술이 아니라 **권고 운영 기준**입니다. 근거 패턴은 raw chat과 기존 guideline에 있습니다.  

### 5.1 채널 목적

이 그룹은 아래 6개만 다룹니다.

1. Vessel SITREP
2. Cargo loading/offloading progress
3. Berth / anchorage / tide / weather exception
4. Manifest / pre-arrival / port clearance
5. FW / MGO / provision / pilot / gate readiness
6. Next port call / next voyage decision

### 5.2 태그 규칙

첫 줄 태그 고정:
`[URGENT][ACTION][FYI][ETA][COST][GATE][CRANE][MANIFEST][RISK]`

JPT71 전용 권장 조합:

* 항차 상태: `[FYI][ETA]`
* berth/tide 지연: `[RISK][ETA]`
* manifest/pre-arrival: `[MANIFEST][ACTION]`
* crane/forklift shortage: `[CRANE][RISK]`
* sailing hold: `[URGENT][RISK]`

### 5.3 첫줄 포맷

아래 형식만 사용:

* `[JPT71]/[VESSEL STATUS]/[ETA-ETD]/[ACTION]`
* `[JPT71]/[CARGO]/[LOADING or OFFLOADING]/[STATUS]`
* `[JPT71]/[PORT]/[BERTH/TIDE]/[RISK]`
* `[JPT71]/[DOC]/[MANIFEST]/[REQUEST]`

예시:

* `[JPT71]/[AGI WEST HARBOR]/[ETA 06:30 GST]/[BERTH STATUS]`
* `[JPT71]/[20MM AGGREGATE]/[OFFLOADING]/[ONGOING]`
* `[JPT71]/[NFP]/[PILOT+BERTH]/[ACTION]`

### 5.4 SITREP 규칙

* **07:30 / 16:00** 2회 고정.
* 필수 항목:

  * current location
  * berth / anchorage
  * ETA / ETD
  * cargo on board
  * tide/weather restriction
  * next port call
  * required support(FW/MGO/manifest/pilot)

### 5.5 D-1 planning 규칙

* **D-1 16:00**까지 다음 항차 확정.
* 반드시 포함:

  * voyage no.
  * cargo type
  * qty / truck qty / bundle qty
  * loading location
  * offloading location
  * equipment
  * manifest owner
  * tide/berth risk
  * next voyage forecast

### 5.6 Tide-window rule

* AGI RORO/LOLO는 **tide-dependent operation**으로 본다.
* `allowable tide`, `high tide suspension`, `resume tomorrow morning`을 사후 보고가 아니라 **사전 risk**로 먼저 올린다.
* group에는 아래를 명확히 적는다:

  * operation start window
  * suspension threshold
  * expected stop time
  * resume time

### 5.7 AGI harbor / Jetty#3 rule

* AGI는 **West Harbor Jetty#3** availability를 최우선 확인.
* berth unavailable 시:

  1. anchorage standby
  2. next call time
  3. radio room coordination status
  4. alternative jetty feasibility
     순으로 공유.
* `captain will try`, `port captain informed` 같은 표현만 남기지 말고 **시간 포함**.

### 5.8 Jumbo bag rule

* MW4 loading 시:

  * GRM trailer arrival time
  * forklift type
  * loader/shovel readiness
  * tarpaulin covering
  * total bags loaded
  * remaining balance
  * expected cast-off
    를 묶어서 공유.
* AGI offloading 시:

  * ALS 145T crane booking 여부
  * day-only / night shift 여부
  * LOLO method
  * complete vs balance bags
    를 반드시 명시.

### 5.9 HCS / NFP / special cargo rule

* HCS, wall panel, STSR, A-frame, crane, excavated soil는 일반 aggregate 항차와 분리.
* 필수 공지:

  * berth confirmation
  * pilot / gate pass / NOC
  * pre-sling status
  * bundle qty
  * draft after loading
  * exact next port
* double stacking 금지, cargo height 확인 필수. 이 패턴은 raw chat에서 실제로 반복됩니다. 

### 5.10 Manifest / pre-arrival rule

* signed manifest 또는 final manifest가 없으면 `waiting manifest` 상태를 명확히 표시.
* pre-arrival notice는 group과 email 양쪽 관리.
* captain이 manifest 미수령 상태에서 entry difficulty를 여러 번 호소하므로, **manifest owner를 명시**한다. 

### 5.11 FW / MGO / provision rule

* fresh water, diesel oil, food provision은 **24시간 전 pre-alert**.
* same-day 요청 금지.
* supply 필요 시:

  * quantity
  * required port
  * required by time
  * gate pass 여부
  * supplier contact
    를 포함.

### 5.12 CEP / vetting / maintenance rule

* CEP / vetting / observation close-out는 **항차 중간 공백시간**에 붙여서 관리하지 말고, 별도 state로 선언.
* 상태값:

  * `Under Maintenance`
  * `Vetting Booked`
  * `CEP Pending`
  * `CEP Renewed`
  * `Operational`
* DAS/Zirku/Ettok call 전에는 반드시 현재 permit status 재확인.
